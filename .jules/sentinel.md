## 2026-03-22 - [CRITICAL] Prevent API Key Bundle Leakage and Parameter Injection
**Vulnerability:** The `vite.config.ts` file was using Vite's `define` configuration to explicitly substitute `process.env.GEMINI_API_KEY` globally during the build process. This bakes the actual API key value directly into the client-side JavaScript bundle, making it accessible to anyone who downloads the code. Additionally, the `GeminiService` was directly concatenating the API key to a URL, leading to potential HTTP Parameter Injection.
**Learning:** For frontend applications implementing Bring Your Own Key (BYOK) architecture via `localStorage`, `process.env` substitution during the build process completely undermines the security posture by leaking secrets from the server's build environment. Also, any un-encoded dynamic input concatenated into fetch request URLs poses injection risks.
**Prevention:** Never use Vite's `define` to substitute sensitive `process.env` variables. Always rely on runtime injection strategies (like fetching from a backend, or strictly reading from `localStorage` for BYOK). Additionally, dynamically injected API keys in fetch request URLs must be wrapped with `encodeURIComponent`.
