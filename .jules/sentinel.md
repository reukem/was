## 2025-03-08 - Vite `define` Exposes Secrets in Bundle
**Vulnerability:** The `vite.config.ts` used `define: { 'process.env.GEMINI_API_KEY': JSON.stringify(env.GEMINI_API_KEY) }`. This causes Vite to statically replace all occurrences of `process.env.GEMINI_API_KEY` in the client-side code with the literal value of the API key from the build environment. This bakes the secret directly into the public minified bundle, exposing it to anyone who inspects the client-side JavaScript.
**Learning:** Vite's `define` option is for creating global constants at build time, not for securely injecting environment variables at runtime. Even if the codebase doesn't reference `process.env.GEMINI_API_KEY` directly right now, having it in the `define` block is a loaded gun. If any code or dependency ever references it, the secret is leaked. The application correctly uses a BYOK architecture reading from `localStorage`, making this `define` block completely unnecessary and extremely dangerous.
**Prevention:** Never use Vite's `define` option with sensitive environment variables (like API keys or passwords) for a client-side application unless you explicitly intend for those values to be public. Rely on secure backend API calls or BYOK (Bring Your Own Key) patterns where users supply their own keys via the UI/localStorage.
