## 2024-05-12 - [Critical] Vite Client-Side Secret Leak via define
**Vulnerability:** The Vite configuration (`vite.config.ts`) was using the `define` block to map server-side environment variables (`env.GEMINI_API_KEY`) to `process.env.API_KEY` and `process.env.GEMINI_API_KEY`. This injects the plaintext value of the API key directly into the client-side JavaScript bundle.
**Learning:** In modern frontend tooling like Vite, the `define` property does static string replacement during the build process. Exposing secrets through it acts the same as hardcoding them in the source code. The project correctly implements a BYOK pattern (`localStorage`), making this injection both unsafe and redundant.
**Prevention:** Never map sensitive server/build environment variables directly into client-side code using `define` or `VITE_` prefixes unless the key is specifically intended to be public (e.g., Firebase public config). Always prefer client-provided configuration (BYOK) for LLM API keys.
