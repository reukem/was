## 2024-05-14 - Prevent Hardcoded Secret Injection in Vite Config
**Vulnerability:** Vite's `define` block in `vite.config.ts` was being used to inject `.env` secrets (`process.env.API_KEY`, `process.env.GEMINI_API_KEY`) directly into the client-side bundle during build time. This completely exposes the backend API keys to any user viewing the minified frontend source.
**Learning:** The application implements a Bring Your Own Key (BYOK) architecture where API configurations strictly rely on `localStorage`. The `define` configuration was not only insecure but also architecturally unnecessary, bypassing the intended local storage mechanic.
**Prevention:** Never use Vite's `define` to map sensitive environment variables to the client window/process object unless explicitly meant for public consumption (e.g., `VITE_PUBLIC_API_URL`). For BYOK architectures, rely solely on runtime injection (like `localStorage.getItem()`) or proxy through a secure backend, and avoid baking secrets into the build entirely.
