## 2026-03-11 - [Hardcoded Secret in Build Config]
**Vulnerability:** The `vite.config.ts` was using the `define` configuration option to globally replace `process.env.GEMINI_API_KEY` and `process.env.API_KEY` with the actual value of `env.GEMINI_API_KEY` at build time. This results in the API key being baked directly into the public client-side JavaScript bundle, allowing anyone to extract and misuse the key.
**Learning:** Vite's `define` config replaces variables with string literals across the codebase during build. While useful for public configuration or build flags, using it to inject sensitive keys into a client-side app exposes those secrets publicly, undermining the intended BYOK (Bring Your Own Key) architecture using `localStorage`.
**Prevention:** Never use build-time injection (like Vite's `define`, Webpack's `DefinePlugin`, etc.) to embed sensitive keys, passwords, or secrets into a frontend build. Ensure any API keys used in the client are fetched securely from a backend endpoint or provided by the user (BYOK). Keep `process.env` logic isolated to server-side code or backend APIs.
