## 2025-02-28 - [Prevent API Key Leakage via Vite Config Define]
**Vulnerability:** The application was using Vite's `define` block to statically replace `process.env.API_KEY` and `process.env.GEMINI_API_KEY` with the `GEMINI_API_KEY` environment variable at build time. This practice embeds the secret API key directly into the compiled, client-side JavaScript bundle, making it easily extractable by anyone inspecting the public code.
**Learning:** For client-side applications (like React SPAs built with Vite), injecting server-side environment variables containing sensitive secrets via build-time substitution exposes those secrets. The application correctly implements a "Bring Your Own Key" (BYOK) architecture via `localStorage` (as seen in `SettingsModal.tsx` and `GeminiService`), making the build-time injection completely unnecessary and dangerous.
**Prevention:** Never use Vite's `define` option (or similar mechanisms like Webpack's `DefinePlugin`) to inject sensitive environment variables (API keys, secrets, passwords) into frontend bundles. Rely on secure backend proxies or, if intentionally designed for client-side API calls, require users to provide their own keys securely (e.g., via BYOK logic).
