## 2025-02-21 - [Remove Exposed Vite Build Environment Variables]
**Vulnerability:** A `define` block in `vite.config.ts` was inappropriately injecting server-side secrets (`GEMINI_API_KEY` from `.env`) into the client-side JavaScript bundle. Any public reference to `process.env.GEMINI_API_KEY` or `process.env.API_KEY` would result in this secret being statically baked into the built files (`dist/`).
**Learning:** Vite's `define` config replaces targeted strings universally across the AST at build-time. This effectively bypasses typical `import.meta.env` mechanisms, creating a covert path for secret leakage when using server-side configuration loaders like `loadEnv`. It is crucial to remember that the codebase inherently relies on `localStorage` for its "Bring Your Own Key" (BYOK) architecture, rendering the global injection completely unnecessary and exclusively dangerous.
**Prevention:** Avoid configuring `define` with `process.env.*` variables in client-side bundling tools like Vite unless there's an irrefutable architectural requirement (and even then, only for non-secrets). Instead, enforce strict reliance on `import.meta.env` for safe, VITE-prefixed variables, or maintain BYOK mechanisms (like `localStorage`) that never cross the build boundary.
