## 2024-05-24 - [HTTP Parameter Injection via API Key]
**Vulnerability:** The Gemini API Key, which is provided dynamically by the user via a settings form and stored in `localStorage`, was being interpolated directly into the API request URL (`https://.../gemini-2.0-flash:generateContent?key=${this.apiKey}`) without any URL encoding. This allowed malicious users or compromised local storage values to inject arbitrary URL query parameters or alter the request structure.
**Learning:** Even fields that seem strictly internal like an API key need to be sanitized or encoded when injected into a URL, because the application utilizes a Bring Your Own Key (BYOK) architecture where the user has direct control over the value. Direct string interpolation into URLs is a common source of HTTP parameter injection.
**Prevention:** Always use `encodeURIComponent()` when dynamically interpolating user-controlled data or data from untrusted sources (like `localStorage`) into URL query parameters.
