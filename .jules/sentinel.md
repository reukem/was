## 2024-05-18 - [CRITICAL] Fix API Key Exposure in Vite Config
**Vulnerability:** The application was exposing a highly sensitive API key (`GEMINI_API_KEY`) to the client-side code bundle by utilizing Vite's `define` config property. This explicitly bypassed secure key management architectures such as passing values at runtime through a backend, or pulling them via a BYOK configuration (like `localStorage`).
**Learning:** Hardcoding or embedding secrets in frontend environments via bundler config directives (e.g., `define: { 'process.env': ... }`) bakes those secrets into static bundles that are directly viewable in the browser. Even if the keys were loaded safely in the Node.js build process via `.env`, exposing them this way negates any backend protection, as anyone with the client bundle could extract them.
**Prevention:** Strictly forbid placing secrets inside `vite.config.ts` or similar client-side bundler configurations. Instead, rely strictly on secure backend APIs to handle sensitive operations or use runtime user-supplied credentials (BYOK) loaded via mechanism like `localStorage` which are kept out of the codebase completely.
