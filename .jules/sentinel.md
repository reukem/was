## 2024-05-24 - [Input Handling & API Security Enhancements]
**Vulnerability:** HTTP Parameter Injection risk in `src/App.tsx` where an unencoded API Key was directly passed into a template literal for `fetch` API call. Also, missing input length validations on chat and settings inputs (`src/App.tsx`, `src/components/SettingsModal.tsx`) which could lead to Client-side DoS, and missing disabled states allowing for concurrent form submissions.
**Learning:** Hardcoded interpolations of user-supplied or locally-stored settings directly into URLs without encoding can lead to parameter injection or malformed requests. Furthermore, client-side input limits and concurrent submission guards are critical for performance and DoS prevention in a React app.
**Prevention:** Always use `encodeURIComponent` for dynamic variables injected into URL parameters. Consistently apply `maxLength` attributes to all `<input>` elements. Manage UI state carefully by disabling form submissions while async operations (like API calls) are in progress.
