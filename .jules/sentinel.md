## 2025-03-01 - [CRITICAL] Prevent API Key leakage through Vite build system
**Vulnerability:** The application was exposing the Gemini API key through Vite's `define` config (`process.env.API_KEY` and `process.env.GEMINI_API_KEY`). While Vite provides `.env` capabilities, injecting secrets via `define` causes the actual string value to be hardcoded into the final client-side JavaScript bundle, making it trivial for any user to extract the key by viewing the source.
**Learning:** Build tools like Vite have mechanisms to inject variables into the client bundle at build time (e.g., `define` or `VITE_*` environment variables). Any secrets included this way are completely exposed to the client. The application is meant to follow a Bring Your Own Key (BYOK) pattern, storing the API key in `localStorage` rather than relying on server-side or build-time injection.
**Prevention:** Never use Vite's `define` or `VITE_*` environment variables for sensitive API keys. Ensure the client application always prompts the user for keys or securely fetches them from a backend, relying exclusively on client-side storage mechanisms like `localStorage` for BYOK implementations.
