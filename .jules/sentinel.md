## 2025-05-20 - [CRITICAL] Vite Build-Time Secret Injection
**Vulnerability:** The `vite.config.ts` file was configured to blindly inject `process.env.API_KEY` and `process.env.GEMINI_API_KEY` into the client-side bundle using the `define` property.
**Learning:** Build tools like Vite replace these `process.env` references with string literals at build time. If an environment variable is present on the build machine (e.g., CI/CD), that secret becomes hardcoded into the public JavaScript files, permanently exposing it to anyone who views the site source.
**Prevention:** Never use `define` or `EnvironmentPlugin` to inject sensitive keys unless they are explicitly intended to be public (like Firebase config). For secrets, use backend proxies or runtime configuration injection that doesn't bake values into static assets. In this app's case, the "Bring Your Own Key" architecture (storing keys in `localStorage`) is the correct pattern, and the build config was undermining it.
