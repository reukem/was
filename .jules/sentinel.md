## 2026-03-31 - Fix Vite Config Secret Leakage
**Vulnerability:** The `vite.config.ts` file was injecting sensitive environment variables (`GEMINI_API_KEY`) into the `define` configuration. This caused Vite's build process to statically replace `process.env.GEMINI_API_KEY` with the actual secret string in the generated frontend JavaScript bundle, completely exposing the server's API key to the public.
**Learning:** Vite's `define` config replaces global constants with string literals at build time. Attempting to pass secrets this way bakes the secrets into the public bundle, unlike Node.js where `process.env` is dynamic at runtime.
**Prevention:** Never use `define: { 'process.env.KEY': ... }` to inject sensitive keys. Instead, use `define` only to safely stub required `process.env` references for third-party libraries with empty strings (e.g., `JSON.stringify('')`) to avoid runtime reference errors, and rely on secure client-side input (BYOK) or backend proxying for sensitive API calls.
